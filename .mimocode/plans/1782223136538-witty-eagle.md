# TournamentHQ Frontend/UX Improvement Plan

## Overview
The frontend has significant CSS/JS duplication across pages, no pagination, limited accessibility, and UX gaps (no loading states, `alert()` validation, silent fetch errors). This plan addresses the highest-impact issues in order of effort vs. benefit.

---

## 1. Extract Shared CSS into `assets/css/public.css`

**Problem**: The `:root` variables, header/nav, body reset, footer, and form styles are copy-pasted across 6 public pages (`index.php`, `login.php`, `play.php`, `organize.php`, `register.php`, `resend-verification.php`).

**Approach**:
- Create `assets/css/public.css` containing:
  - `:root` CSS custom properties (currently duplicated in 6 files)
  - `.site-header`, `.nav-container`, `.brand-logo`, `.nav-links`, `.nav-link-item` (4 files)
  - `body` reset styles (6 files)
  - `.site-footer` (3 files)
  - `.login-page`, `.login-card`, `.login-sep`, `.input-wrap`, `.input-icon`, `.show-pw` (3 files)
  - Sport/category badge styles (currently inline `style=` in 4 files)
- Replace inline `<style>` blocks in public pages with `<link rel="stylesheet" href="assets/css/public.css">`
- Remove duplicate `:root` declarations and redundant resets from each page
- Consolidate `resend-verification.php`'s custom `.card`, `.button`, `.alert-*` to reuse existing classes from `style.css`

**Files to modify**:
- Create: `assets/css/public.css`
- Modify: `index.php`, `login.php`, `play.php`, `organize.php`, `register.php`, `resend-verification.php`

---

## 2. Extract Shared JS into `assets/js/public.js`

**Problem**: `lucide.createIcons()` init, `togglePassword()`, and modal fallback logic are duplicated across pages.

**Approach**:
- Create `assets/js/public.js` containing:
  - Lucide icon init (`lucide.createIcons()` on DOMContentLoaded)
  - `togglePassword(inputId)` utility function
  - Light init for any shared behavior
- Each public page includes `<script src="assets/js/public.js"></script>` instead of inline duplicated code
- Remove inline `togglePassword()` from `login.php` and `register.php`
- Remove duplicate Lucide init blocks from all 5 public pages

**Files to modify**:
- Create: `assets/js/public.js`
- Modify: `index.php`, `login.php`, `play.php`, `organize.php`, `register.php`

---

## 3. Add Shared UI Components to `main.js`

**Problem**: Generic utilities (collapse, toast, validation, loading) are either missing or duplicated in page-specific scripts.

**Approach** — add these to `assets/js/main.js`:

| Utility | Purpose |
|---------|---------|
| `TTMS.toast(message, type)` | Lightweight toast notifications for async feedback (replaces `alert()`) |
| `TTMS.loading(button, state)` | Add/remove loading spinner on form buttons during submission |
| `TTMS.validate(form)` | Simple inline validation (check required fields, show error below input) |
| `TTMS.debounce(fn, ms)` | Debounce utility for search inputs |
| `TTMS.setupCollapse()` | Generic accordion/collapse (extracted from `umpire/dashboard.php` lines 512-553) |
| `TTMS.copyToClipboard(text, btn)` | Copy tournament links/access codes |

**Files to modify**:
- `assets/js/main.js` (add new utilities)
- `player/index.php` (replace `alert()` validation with `TTMS.validate()`)
- `register.php` (replace `alert()` validation)
- `umpire/dashboard.php` (extract `setupCollapse()` to `TTMS`, replace `alert()`)
- `includes/bracket_view.php` (add hover effects via CSS instead of inline `onmouseover`)

## 4. Add Pagination

**Problem**: No page in the application has pagination. Lists of players, tournaments, users, and matches grow unbounded.

**Approach** — Server-side pagination with reusable pattern:

- Add a `paginate($page, $perPage, $total)` helper to `includes/auth.php` or a new `includes/helpers.php`
- Add `LIMIT ?, ?` to all list queries with `?page=` URL parameter
- Create a reusable pagination control partial (`includes/pagination.php`)
- Apply to these pages (ordered by data growth risk):

| Page | Function to modify | Current query |
|------|-------------------|---------------|
| `admin/manage_players.php` | `getAllPlayers()` | `SELECT * FROM players ...` (no limit) |
| `admin/manage_users.php` | inline query | `SELECT * FROM users ...` (no limit) |
| `admin/manage_tournaments.php` | `getAllTournaments()` | `SELECT * FROM tournaments ...` (no limit) |
| `organizer/players.php` | inline query | `SELECT * FROM players ...` (no limit) |
| `organizer/tournaments.php` | inline query | `SELECT * FROM tournaments WHERE organizer_id=...` |
| `organizer/index.php` | inline queries | Multiple unbounded lists |

- Default 20 items per page, with `?page=N` query parameter
- Show "Page X of Y (N results)" with prev/next and page number links

**Files to modify**:
- Create: `includes/pagination.php`
- Modify: `includes/helpers.php` (add `paginate()`), `modules/tournaments/tournament_functions.php` (add LIMIT to `getAllTournaments`), `modules/players/player_functions.php` (add LIMIT to `getAllPlayers`), 6 admin/organizer pages

## 5. Accessibility Quick Wins

**Problem**: Only 2 `aria-label` attributes exist in the entire codebase. No focus management, no skip link, poor color contrast.

**Approach**:

| Fix | Where | Details |
|-----|-------|---------|
| Add `aria-label="Toggle sidebar"` | `includes/header.php` line 133 | On sidebar toggle button |
| Add `aria-expanded` toggle | `assets/js/main.js` `initSidebar()` | Toggle on sidebar open/close |
| Add Escape key to close sidebar | `assets/js/main.js` `initSidebar()` | Keyboard users can close sidebar |
| Add `aria-hidden="true"` on closed modals | `assets/js/main.js` `openModal()`/`closeModal()` | Toggle `aria-hidden` |
| Add focus trap in modals | `assets/js/main.js` `openModal()` | Trap Tab key within modal |
| Add `:focus-visible` styles | `assets/css/style.css` | Visible outline for keyboard navigation |
| Add skip-to-content link | `includes/header.php` + `assets/css/style.css` | `<a href="#main-content" class="skip-link">Skip to content</a>` |
| Fix color contrast for `--text-400` | `assets/css/style.css` line 35 | Change `#6065a0` to `#7a7fbf` (4.6:1 ratio) |
| Add `<label>` for unlabeled inputs | `player/index.php`, `umpire/dashboard.php` | Search filters, form inputs |

**Files to modify**:
- `includes/header.php`, `assets/js/main.js`, `assets/css/style.css`, `player/index.php`, `umpire/dashboard.php`

## 6. UX Polish

**Problem**: No loading states, `alert()` validation, silent fetch errors, inconsistent confirm dialogs.

**Approach**:

| Improvement | Implementation |
|-------------|---------------|
| Replace `alert()` with inline validation | Use `TTMS.validate()` from step 3; show error text below input fields |
| Add loading spinners on form submit | Use `TTMS.loading()` from step 3; disable button + show spinner |
| Show fetch errors in UI | In `main.js`, replace `.catch(() => {})` with `.catch(() => TTMS.toast('Error occurred', 'error'))` |
| Consistent confirm for destructive actions | Use `data-confirm` attribute uniformly (already supported by `main.js`) |
| Manual flash dismiss | Add click handler to flash `x` button (currently only auto-dismiss) |
| Replace inline JS hover effects with CSS | Move `onmouseover`/`onmouseout` handlers to CSS `:hover` rules |

**Files to modify**:
- `assets/js/main.js`, `assets/css/style.css`
- `player/index.php` (replace alert validation, add hover CSS)
- `register.php` (replace alert validation)
- `umpire/dashboard.php` (replace alert, add collapse to main.js)
- `includes/bracket_view.php` (replace inline hover JS)

## Verification

After each phase:
1. Open every modified page in browser — verify visual appearance matches before changes
2. Test all role dashboards (admin, organizer, player, umpire)
3. Test on mobile viewport (Chrome DevTools, 375px width) — verify responsive behavior
4. Test keyboard navigation — Tab through forms, Escape to close sidebar/modals
5. Check browser console for JS errors
6. For pagination: test page 1, middle page, last page, and empty results
7. Run any existing PHP syntax checks if available
