# Fix: Knockout Stage Bracketing — Rank / Same-Group Bugs

## Bugs Found

### Bug 1: Rank 2 vs Rank 2 (and Rank 1 vs Rank 1) Matchups

In `modules/tournaments/bracket_functions.php`, the `generateKnockoutStage` function (line 340) has a seeding bug that allows **rank 2 vs rank 2** (and potentially rank 1 vs rank 1) first-round matchups when the number of groups is not a power of 2 (e.g., 3, 5, 6, 7 groups).

**Root Cause (lines 358–376):** The code puts ALL rank 1s first, ALL rank 2s second, then pads byes only at the end. This causes rank 2 players to leak into the "top half" of the bracket, breaking the standard bracket seeding guarantee.

**Example with 3 groups (6 qualifiers):**
```
seeded = [A_R1, B_R1, C_R1, A_R2, B_R2, C_R2, null, null]
```
After standard bracket seeding `[1,8,4,5,2,7,3,6]`, first-round pairs:
- A_R1 vs null (bye)
- **A_R2 vs B_R2** ← BUG: R2 vs R2
- B_R1 vs null (bye)
- **C_R1 vs C_R2** ← BUG: Same group matchup

### Bug 2: Same-Group Opponents Can Face Each Other

The current code can also pair rank 1 and rank 2 from the **same group** in the knockout first round. This is undesirable — teams from the same group should be separated in the knockout bracket.

**Example with 3 groups:** C_R1 vs C_R2 (same group, different rank) appears as a first-round matchup.

## Fix

**File:** `modules/tournaments/bracket_functions.php`  
**Lines to change:** 355–376

Replace the current seeding logic:

```php
// CURRENT (buggy):
$rank1s = [];
$rank2s = [];
for ($i = 0; $i < $count; $i++) {
    if ($i % 2 === 0) {
        $rank1s[] = $qualifiers[$i];
    } else {
        $rank2s[] = $qualifiers[$i];
    }
}
$seeded = array_merge($rank1s, $rank2s);

// Pad to next power of 2 with null entries (byes)
$bracketSize = 1;
while ($bracketSize < count($seeded)) {
    $bracketSize *= 2;
}
while (count($seeded) < $bracketSize) {
    $seeded[] = null; // bye slot
}
```

With this fixed version:

```php
// FIX: Calculate bracket size first, then pad R1s and R2s into their own halves
$bracketSize = 1;
while ($bracketSize < $count) {
    $bracketSize *= 2;
}

$rank1s = [];
$rank2s = [];
for ($i = 0; $i < $count; $i++) {
    if ($i % 2 === 0) {
        $rank1s[] = $qualifiers[$i];
    } else {
        $rank2s[] = $qualifiers[$i];
    }
}

$halfSize = $bracketSize / 2;
$seeded = array_merge(
    array_pad($rank1s, $halfSize, null),  // Top half: R1s + byes
    array_pad($rank2s, $halfSize, null)   // Bottom half: R2s + byes
);
```

**Why this works:** The standard bracket seeding always pairs seed `k` with seed `(B+1-k)` in the first round. By placing all rank 1s in the top half (seeds 1 to B/2) and all rank 2s in the bottom half (seeds B/2+1 to B), every first-round match is guaranteed to be R1 vs R2. Byes land in whichever half has fewer qualifiers.

**Same-group separation:** Since R1 from group X is at seed 1 (top half) and R2 from group X is at seed B/2+1 (bottom half), they would only be paired if `1 + (B/2+1) = B+1`, which simplifies to `B=2` (1 group only — unavoidable). For 2+ groups, same-group opponents are automatically separated.

Also update the comment on lines 355-357 to reflect the new logic.
