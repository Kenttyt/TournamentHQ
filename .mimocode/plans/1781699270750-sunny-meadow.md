# Login Page for TournamentHQ

## Goal
Create a standalone `login.php` page and update `play.php` to link to it.

## Files to Modify/Create
- `C:\xampp\htdocs\TournamentHQ\login.php` (NEW)
- `C:\xampp\htdocs\TournamentHQ\play.php` (update Login link)

## Implementation

### 1. Create `login.php`
Full-page login form with the same dark theme and CSS variable system as other pages.

**Structure:**
- Header: TournamentHQ logo (links to index.php) — NO nav links (Play/Organize removed per rules)
- Centered login card with:
  - Headline: "Welcome back"
  - Lead text: "Sign in to your TournamentHQ account"
  - Username input with icon
  - Password input with show/hide toggle
  - "Sign In" button (purple gradient)
  - "Forgot password?" link
  - Divider: "Don't have an account?"
  - "Create Account" link → registration page (placeholder #)
- Footer: "TournamentHQ"

**PHP Logic:**
- Include `auth.php`
- If already logged in → redirect to dashboard
- Handle POST form submission using `loginUser()` from auth.php
- On success → redirect to dashboard by role
- On failure → show error message
- CSRF token generation/validation

**CSS:**
- Reuse the login card styles from index.php (`.login-card`, `.input-wrap`, `.btn-login`, etc.)
- Same `:root` CSS variables
- Responsive: card centers on mobile

### 2. Update `play.php`
Change Login button href from `/TournamentHQ/index.php` to `/TournamentHQ/login.php`.

## Verification
1. Open play.php — click Login → navigates to login.php
2. login.php shows login form with username/password fields
3. Submit valid credentials → redirects to dashboard
4. Submit invalid credentials → shows error message
5. Already logged in → redirects to dashboard
6. Logo links back to index.php
