# Plan: Fix Admin Password Reset Bug

## Problem
`config/database.php:202-284` — `ensureDefaultAdminUser()` runs on every database connection and resets the admin password back to `admin` if it's been changed. Line 230: `$needsPassword = !password_verify('admin', $admin['password'])` triggers a password reset whenever the current password isn't 'admin'.

## Fix
Change `ensureDefaultAdminUser()` to only **create** a default admin if no admin user exists at all. Never reset an existing admin's password, username, or email.

### File: `config/database.php`

Replace the `ensureDefaultAdminUser()` method (lines 201-284) with:

1. Check if any user with role='admin' exists
2. If YES → return immediately (admin already exists, don't touch anything)
3. If NO → create a default admin with username='admin', password='admin', email='admin@ttms.com'
4. Remove the `static $done` flag (no longer needed — the method is now idempotent and fast)

This preserves the original intent (ensure there's always an admin) while eliminating the password reset behavior.

## Verification
1. Change the admin password to something else
2. Reload any page
3. Verify the password is NOT reset back to 'admin'
4. Verify the admin user still exists and can log in
