# Security Improvements — Critical Fixes Only

## 1. CSRF Protection

**Problem:** No CSRF tokens in any form. All POST forms vulnerable to cross-site request forgery.

**Fix in `includes/auth.php`:**
- Add `generateCsrfToken()` — creates a random token, stores in `$_SESSION['csrf_token']`, returns it
- Add `validateCsrfToken()` — compares `$_POST['csrf_token']` against session token

**Fix in every POST form (~15 files):**
- Add `<input type="hidden" name="csrf_token" value="<?= csrfToken() ?>">` to each form
- Add `if (!validateCsrfToken()) { ... exit; }` at the top of each POST handler

**Files:**
- `includes/auth.php` — add 2 helper functions
- `login.php` — 2 forms (login + register)
- `register.php` — 1 form
- `admin/manage_users.php` — 4 forms (create, edit, toggle, delete) + inline forms
- `admin/manage_tournaments.php` — create, edit, delete, status change forms
- `admin/manage_players.php` — forms
- `admin/bracket_generator.php` — bracket generation form
- `organizer/tournaments.php` — create, edit, delete, status, approve/reject forms
- `organizer/umpire_access.php` — generate, revoke forms
- `organizer/players.php` — forms
- `organizer/bracket_generator.php` — bracket generation form

## 2. Session Fixation Prevention

**Problem:** Session ID not regenerated after login, allowing session hijacking if attacker knows session ID before login.

**Fix in `includes/auth.php` `loginUser()`:**
- After setting `$_SESSION` vars, call `session_regenerate_id(true)`

**Fix in `google-callback.php`:**
- After auto-login (line 98-101), call `session_regenerate_id(true)`

---

## 3. Error Message Information Leakage

**Problem:**
- `register.php:87` — `'Registration failed: ' . $e->getMessage()` leaks PDO errors
- `config/database.php:35` — `die(json_encode(['error' => 'Database connection failed: ' . $e->getMessage()]))` leaks DB errors
- `verify.php:48` — `'Verification failed: ' . $e->getMessage()` leaks exception details

**Fix:**
- `register.php:87` → `$error = 'Registration failed. Please try again.';`
- `config/database.php:35` → `error_log('TournamentHQ DB Error: ' . $e->getMessage()); http_response_code(500); die(json_encode(['error' => 'Service temporarily unavailable.']));`
- `verify.php:48` → `setFlash('error', 'Verification failed. Please try again.');`

---

## 4. XSS via Unescaped Error Output

**Problem:** In `login.php`, error messages are output without escaping:
- Line 398: `<?= $loginError ?>` — can contain intentional HTML (`<a>` tag on line 96)
- Line 461: `<?= $regError ?>` — always plain text but unescaped

**Fix in `login.php`:**
- For `$regError` (line 461): use `<?= e($regError) ?>` since it's always plain text
- For `$loginError` (line 398): introduce a `$loginErrorIsHtml` flag; when true, output as-is (for the email verification link case); when false, escape with `e()`. Default to escaped.
